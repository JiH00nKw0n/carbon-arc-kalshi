# Factor 1 — 두 번째 (X·Z·Y) 케이스 제안: "결합이 일반화되는가"

> 이 문서는 **제안서**다. Factor 3(카드 × 어닝콜 → 매출 서프라이즈)에서 확립한 "정형 대체데이터(X)와 자연어(Z)를 LLM으로 결합하면 성과(Y)를 더 잘 예측한다 — 부분의 합을 넘는 상호보완(synergy)이 있다"는 결과를, **다른 대체데이터로 한 번 더** 보이기 위한 다음 케이스를 고른다.
> 변수 표기·검정 방법론은 [`../factor3/README.md`](../factor3/README.md), [`../factor3/DESIGN.md`](../factor3/DESIGN.md)와 동일하게 따른다.

## 0. 왜 Factor 1인가 (맥락)

2026-06-24 리서치 채널 합의:

- **이용재(Lawrence):** "종목보다는 **case를 늘리는 게** 중요. '대체데이터를 텍스트와 LLM으로 보강할 수 있다'를 주장하려면 **다양한 대체데이터 + 추가데이터의 조합**을 보여주는 게 좋다. 단 X와 Z 조합이 여러 케이스로 늘면 **비용·시간 관리**가 필요."
- **결정(Jihoon):** 이번 주는 **더 다양한 case**를 찾는다 (cc. Sumin Kim).

→ 즉 Factor 1의 목표는 *절대 예측력 신뢰구간을 좁히는 것(=종목 확장)*이 **아니라**, F3의 결론이 **카드가 아닌 다른 대체데이터에서도 재현되는가(=일반화)**를 보이는 것이다. "한 번은 우연, 두 번은 패턴"의 두 번째 데이터점.

## 1. 패러다임 복습 (F3에서 고정된 것)

- **X** = 정형 대체데이터, **Z** = 자연어(어닝콜), **Y** = 성과 지표(컨센서스 대비 **서프라이즈 = 잔차**).
- LLM 아키텍처 **R** ∈ {A: 텍스트→점수→OLS, B: end-to-end 직접예측, C: LLM joint feature→OLS}. F3에서 **B(end-to-end)만** 신호를 잡았다(점수 증류는 정보 손실).
- 핵심 주장 = **상호보완(synergy)**: corr(X+Z) > corr(X) + corr(Z). 검정 = shuffle-company surrogate(load-bearing) + company-clustered bootstrap + nested ΔR²/CW/DM-HLN + BH-FDR.
- 누설 통제: 평가는 모델 컷오프 **이후** 발표 분기만, 입력은 전부 point-in-time.

**F3 헤드라인:** 카드만 +0.136, 텍스트만 +0.005, **카드+텍스트 +0.281** (synergy +0.214, p=0.010). 단 절대 예측력 CI는 0 포함.

## 2. "좋은 두 번째 케이스"의 조건

다음 5개를 만족할수록 Factor 1로 강하다:

1. **X의 품질이 검증됨 + 회사단위로 쪼개짐** — Carbon Arc 데이터 품질이 확인되고, **티커/회사 단위 해상도**가 있어야 한다(F3는 종목별 X 필수). (Lawrence 트라이얼 리포트 기준 *Useful* = Card·Web·Foot·Jobs. POS·App·Tech는 conditional/제외.) 소비 7개 밖의 데이터(claims·gov·ads 등)는 §3에서 `Granularity` 필드로 회사단위 가능 여부를 **직접 확인** — 안 되면(예: Freight=레인, Housing=Zip) 패러다임 부적합.
2. **X 단독으로는 Y에 약함/모호함** — 그래야 "텍스트가 맥락을 채워 살아난다"는 synergy가 극적으로 드러난다. (역설적으로 *단독으로 약한* 데이터가 데모로는 최고.)
3. **Z가 X의 모호함을 풀어주는 경제적 메커니즘이 명확** — 텍스트가 왜 그 숫자를 해석해주는지 한 줄로 설명 가능.
4. **Y가 깨끗한 잔차(서프라이즈)** — 컨센서스가 존재해 (actual − consensus)로 측정 가능.
5. **저비용·재사용** — 기존 F3 파이프라인 재활용 + 데이터 추가구매 최소. (공유 promo 잔액 ~$9,194, 다 가능하나 빠르고 싸게.)
6. **F3와 구분됨** — 가능하면 *X만* 바꾸고 Z·Y는 고정 → "대체데이터 채널이 바뀌어도 패러다임이 성립"이라는 **가장 깨끗한 일반화 증거**.

## 3. 가능 후보 X–Y 페어 (Tentative)

> 기준선 **#0 = Card → 매출 (= F3, 검증 완료)**. Z는 별도 표기 없으면 **어닝콜**.

**탈락 (회사단위 불가 → 패러다임 부적합):**

**(별도 축) Z 교체:** 카드(X 고정) × **전문가콜/뉴스(LinqAlpha 자산)** → 매출 도 가능하나, Lawrence 요청(*다양한 X*)과는 결이 달라 후순위.

**추천 묶음:** 빠른 일반화 = **#1 Web(채널)** + **#3 Jobs(타깃)**; 임팩트 큰 한 방(섹터 점프) = **#2 Pharma Claims**.

## 4. 추천: **Factor 1 = #1 (Web 트래픽 × 어닝콜 → 매출 서프라이즈)**

**이걸 먼저 하는 이유:**

1. **synergy 데모가 가장 극적.** Lawrence가 *이미* "웹 트래픽은 매출 프록시가 아니다(단독 YoY 유의성 상실)"를 정량 확인해줬다. 즉 **X 단독 = null이 거의 보장**됨. 여기서 X+Z가 살아나면 "단독으로 무력한 top-of-funnel 신호조차 LLM이 텍스트 맥락을 붙이면 매출 예측이 된다"는, F3보다 **더 강한 일반화 메시지**가 된다.
2. **F3와 가장 깨끗하게 비교됨.** Z(어닝콜)·Y(매출 서프라이즈)·평가표본·재무 baseline을 **그대로 두고 X만 카드→웹**으로 교체 → 차이가 오직 "대체데이터 채널"에서만 옴. 과학적으로 한 변수만 움직인 대조실험.
3. **가장 싸고 빠름.** 웹은 7개 중 최저가 데이터셋. 온라인 종목 30~40개 subset이면 수십 달러. 파이프라인은 F3 `s_aj_ablation.py` 구조를 거의 그대로 재사용.
4. **유니버스가 자연히 갈림.** **온라인/이커머스 종목**(Chewy·Wayfair·eBay·Carvana·Etsy·Amazon…)은 foot traffic이 비어 있어 **웹이 1차 수요 신호**다. F3의 omni-channel 35종목과 겹치지 않는 새 universe → "다양한 case"에도 부합.

**Factor 1 변수 고정안:**
- **X** = CA0030 Web Traffic, 종목별 월간(또는 주간), YoY 가공.
- **Z** = 직전 분기 어닝콜 전사 (F3 매핑 로직 재사용).
- **Y** = 매출 서프라이즈 = (actual − FactSet point-in-time consensus)/consensus.
- **R** = B(end-to-end) 메인, A/C는 보조 비교 (F3 결론 재확인용).
- **표본** = 온라인 수요 종목 ~30개 × post-cutoff 분기 (모델 컷오프 이후).

## 5. 로드맵 (Factor 2 / 3-analog)

Lawrence가 "다양한 조합"을 원했으니, #1 성공 후 난이도·novelty 순으로:

- **Factor 2 = #3 (Jobs × 어닝콜 → 마진/opex 서프라이즈)** — Sumin과 함께. **Y를 매출→비용/마진으로** 바꾸는 케이스라 novelty가 가장 높다. F3 DESIGN이 의도적으로 빼둔 `MarginCostRisk` 계열 점수가 여기서 부활한다("Y=EPS/마진 확장 시 부활"). 채용 의도의 모호함(확장 vs AI 자동화)을 콜이 푸는 구조 — Lawrence의 `/ir-prep` 예시와 동일.
- **섹터 점프 = #2 (Pharma Claims × 어닝콜 → 약품·제품 매출)** — 소비재 밖으로 나가는 가장 강한 일반화. GLP-1처럼 단일약이 매출을 끄는 이름(NVO·LLY)이면 거의 카드급으로 깨끗. claims는 약품·제조사 단위라 **제조사→티커 매핑**이 선행 작업.
- **Factor 3-analog = #4 (Foot × 어닝콜 → SSS/매출)** — 물리 수요 채널. 식당(CMG·SBUX·MCD·DPZ·WEN) universe가 깨끗. foot이 가장 비싸 우선순위는 뒤.

케이스가 모이면 주장: **"카드(F3)·웹·채용·약품claims·방문 어느 대체데이터든, 텍스트+LLM 결합이 단독을 넘어선다"** — 채널 일반화(카드·웹·약품) + 타깃 일반화(매출→마진) 둘 다 커버.
