# test_2026-06-04 — ICL regression: does Claude Opus 4.8 learn card→revenue in-context?

> `scripts/auto/s_ad_icl_regression.py` · results `outputs/auto/icl_regression_results.csv` ·
> plot `outputs/auto/icl_mse_vs_shots.png`

## Setup

Treat card→revenue as an **in-context-learning (ICL) regression**: give Opus 4.8 `k` example pairs
`(card_yoy, rev_yoy)` in the prompt, ask it to predict held-out `rev_yoy`, measure MSE vs `k`.
- **Clean eval via Jan-2026 training cutoff**: examples from **PRE** (`FE_FP_END < 2026-01`, **263** pairs);
  test = **POST** (`≥ 2026-01`, **49** pairs = 2026Q1/Q2) → answers can't be memorized.
- Both as YoY ×100 (%), winsorized 1/99. PRE corr(card,rev)=0.50.
- k grid `0,1,4,16,64`; R=5 random example draws per k (1 for k=0); async (asyncio.gather, conc=8).
- model `claude-opus-4-8`, no temperature (deprecated for this model → some run-to-run noise).
- **Arms**: `anon` (numeric only, `x=.., y=..`) = pure ICL; `labeled` (`WMT 2025Q2: card_spend_yoy=..,
  revenue_yoy=..` + "equity revenue nowcaster" role) = ICL **+ world knowledge**; `synthetic` = positive
  control with a known law `y=0.6x+ε`.
- Baselines (code): OLS(x→y) on the k examples (the "right" linear fit) and predict-mean.

## Results — MSE on POST (lower = better)

| arm | k=0 | k=1 | k=4 | k=16 | k=64 | OLS@64 | mean-base |
|---|---|---|---|---|---|---|---|
| **synthetic** (control) | 14.1 | 9.3 | 15.9 | 8.2 | 11.6 | **5.8** | ~18 |
| **anon** (pure numeric ICL) | 90.6 | 106 | 124 | 84 | **67** | 49 | ~65 |
| **labeled** (ICL + world knowledge) | **33.2** | 43.6 | 43.6 | 37.2 | **31.1** | 51 | ~62 |

(corr(ŷ,y): synthetic ~0.7-0.82; anon ~0.3-0.5; labeled ~0.68-0.79)

## Reading it

1. **Harness validated (synthetic).** With a real linear signal, ICL MSE falls from 14 → ~8-12 and sits
   near the OLS floor (~6), far below predict-mean (~18). Opus genuinely does in-context regression. (Curve
   is noisy, not perfectly monotonic — no temperature control + few draws.)

2. **Pure numeric ICL on real card→revenue is weak.** anon MSE is high (67-124) and only loosely declines
   with k (90→67 by k=64), roughly tracking OLS but hovering around the predict-mean baseline (~65). Why:
   the real card→revenue signal is weak (OLS itself, MSE 49, barely beats mean 65). There isn't much linear
   signal to learn from the numbers alone.

3. **⭐ World knowledge dominates (labeled).** Giving ticker+quarter labels + the "nowcaster" role, Opus
   predicts 2026 revenue YoY with **MSE ~31-44 and corr ~0.7-0.8 — better than OLS (~51) and predict-mean
   (~62), and far better than pure-numeric anon.** Strikingly, **k=0 (no examples) already scores MSE 33,
   corr 0.79.** With company identities, the model brings prior knowledge of each firm's revenue-growth
   regime (and uses the card hint), which beats fitting a regression on card alone.
   - Not leakage: 2026 revenue is post-cutoff. It's *extrapolation from pre-cutoff company knowledge* +
     the provided card feature — these are stable large-caps whose YoY growth is partly predictable.

4. **k-effect differs by arm.** synthetic: MSE↓ with k (textbook). anon: weak ↓. labeled: ~flat — priors
   already saturate at k=0, so extra examples add little.

## Takeaways
- Opus 4.8 **can** do ICL regression (synthetic proves it), and recovers ~the OLS fit from numbers.
- For card→revenue specifically, **the numeric relationship is too weak for pure ICL to shine** — same
  conclusion as every classical test (card carries a small, noisy revenue signal).
- **The model's world knowledge about these companies is a stronger revenue nowcaster than the card→revenue
  regression itself** — labeled-0-shot (priors only) beats 64-shot pure-numeric. The value is in the
  semantic context, not the card numbers.
- Caveat: no temperature control (deprecated) → curves are noisy; k=1 is unstable (one example → wild
  extrapolation; first run had a parse blow-up at k=1). For publication-grade curves, average more draws
  and/or add denser k.
